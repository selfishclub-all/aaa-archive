```
매일 아침 10시 n8n크론
  ↓

  ↓
```

9개 알림톡 중 **직접 입력하는 건 딱 3개**(줌링크, 쿠폰, 라이브혜택). 
	나머지 6개는 앞에 입력한 정보를 자동 재사용. 공유회 1건당 9번 수동 카피 → **3번 터치로 전부 발송**.

#### 문제점 : 아침 10시 크론이 안도는 상황 -> 그래서 오늘 스케쥴이 있는데도 슬랙에 안옴

"매일 10시에 크론 후 발송" 세팅 → 안 옴. 클로드한테 "안 왔어"라고 전달 
→ Claude가 이것저것 확인하다가 "1분마다로 한번 해봐" → 해보니 그건 됨. **특정 시간 지정만 안 먹히는 상황.**

여기서 Claude가 "서버 시간대가 한국이 아닐 수 있다"라고 얘기한 후, 그다음부터 삽질 시작 
— **타임존을 하나씩 대입하며 추측**. UTC로 넣어보고 → 안 됨 → UTC+9 → 안 됨 → UTC-4 → 됨. 3번 만에 맞춤(시간 엄청 씀)

#### 💡 인사이트(클로드가 삽질할 때)

**돌아보면:** n8n에서 `{{$now}}` 한 줄 찍으면 서버가 지금 몇 시라고 생각하는지 바로 나옴. 추측 대입 대신 **직접 확인**해봤으면 끝날 상황

**그럼 내가 어떻게 했어야 클로드가 삽질을 안할 수 있었을지?**
근데 나는 "안 왔어" 이외에 할 수 있는 게 없었음. 
클로드는 "서버 시간 확인해봐"라는 라는 질문을 하면 빨랐을거라고 함 
????질문 자체가 머릿속에 없음. 서버 타임존이라는 개념을 모르는데 어떻게 그걸 물어보는지? 
Claude가 다 세팅한 거고, 나는 그냥 "10시에 온다고 했는데 안 왔어"가 아는 전부. 
"1분마다 해봐"도 Claude가 시킨 거지 내가 생각해낸 게 아님.

**결국**
기술적으로 맞는 질문을 할 필요 없음. "서버 타임존 확인해봐"라는 질문은 그 개념을 아는 사람만 할 수 있고 그건 Claude가 알아서 확인해야 하는 영역.

내가 할 수 있는 건 **"얘가 지금 헛돌고 있다"를 알아채는 것**. Claude가 이것저것 바꿔가면서 계속 시도하는데 계속 안 되면 -> 그때 **"지금 접근이 맞아?"** 한 마디면 된다고함. 
기술을 몰라도 "삽질 패턴"은 보임. 계속 뭔가 넣어보고 안 되고, 또 넣어보고 안 되면? 
-> 그건 추측하고 있다는 신호. **그때 끊어주면 Claude가 전략 자체를 바꿈.**

**하네스 엔지니어링 관점:**
이 삽질의 본질은 **"추측 vs 확인"**. Claude가 타임존을 하나씩 대입하며 추측하고 있었는데, 서버한테 직접 물어보면 1초에 끝나는 일. 이건 타임존만의 문제가 아님

AI가 뭔가를 반복 시도하면서 안 될 때, **"지금 추측하는 거야? 확인하는 거야?"**라는 질문 하나가 삽질을 끊어주는 것 같음. 기술적인 질문을 할 줄 몰라도, **AI가 빙빙 도는 패턴을 잡아채는 것** 자체가 중요

**"안 돼" → "뭐가 되고 뭐가 안 되는지 관찰(내가)" → Claude가 원인 추측 → "그거 확인하는 방법 알려줘"**

**다른 상황에서
- 카피를 계속 고치는데 계속 어색함 → "톤 자체가 틀린 거 아니야?" → Claude가 구조부터 다시 잡음
- 에러를 고쳤는데 또 다른 에러가 나옴 → "하나씩 고치는 게 맞아? 근본적으로 뭔가 잘못된 거 아니야?" → Claude가 원인부터 다시 봄
- 디자인을 이리저리 바꾸는데 마음에 안 듦 → "이 방향 자체가 아닌 것 같아. 아예 다른 식으로 해봐" → Claude가 접근을 바꿈
- "A한테는 가는데 B한테는 안 감" -> "A랑 B 데이터 차이 확인하는 방법?"
- "어제는 됐는데 오늘 안 됨" -> "어제랑 오늘 사이에 뭐가 바뀌었는지 확인하는 방법?"

공통점: **뭐가 잘못인지 몰라도 "지금 안 되고 있다"는 건 알 수 있음.** 그걸 말해줄 것

	- **코드 안전성 우선 튜닝** 때문에, 한 부분 고치면 다른 부분은 건드리기 싫어함 → CSS 전체 밸런스를 못 잡음

	- 인라인 스타일 난장판, 클래스명 충돌 등등 -> 대신 제미나이는 클래스명 충돌나서 카피깨질때도 있음
- **Gemini 웹: 요청을 확장 해석해서 장식까지 덧붙인다**
	- 웹은 결과물이 바로 렌더링되니까 이터레이션이 빠름 -> 하지만 클로드코드는 프리뷰
- 결론
	클로드코드? 못하는 게 아니라 비효율적인 거
	- 이미지를 "본다"와 "CSS로 정확히 재현한다"는 다른 문제
	클로드 코드로도 할 수 있겠지만 근데 **같은 결과를 내려면 2~3배 더 구체적인 지시가 필요**하고, 시간이 더 걸리기 때문에 **비주얼 디자인 이터레이션은 클로드코드 웹이나 제미나이에서 작업하는 편**
