# AAA 홈페이지 & 어드민 — 어떻게 만들었고, 어떻게 돌아가는가

---

## 한 줄 요약

**Obsidian에 마크다운을 쓰면, 자동으로 웹사이트가 만들어진다.**
**어드민 페이지에서 GUI로 수정하면, 자동으로 반영된다.**

---

## 1. 전체 그림 — "마크다운에서 웹사이트까지"

![어드민구조_1.png](/images/어드민구조_1.png)

```
Obsidian Vault (마크다운 작성)
       ↓
  sync-content.sh (이미지 경로 변환, 위키링크 변환)
       ↓
  Astro 프로젝트 (정적 사이트 생성기)
       ↓
  GitHub에 push
       ↓
  Vercel이 감지 → 자동 빌드 → 배포 완료
```

### 핵심 포인트
- **서버가 없다.** 데이터베이스도 없다. 마크다운 파일이 곧 데이터베이스.
- **GitHub가 저장소** 역할을 하고, **Vercel이 배포** 역할을 한다.
- 코드를 push하면 1~2분 안에 사이트가 업데이트된다.

---

## 2. 웹사이트 ↔ 어드민 관계

![어드민구조_2.png](/images/어드민구조_2.png)

홈페이지 URL: **aaa-homepage.vercel.app**

| 페이지 | 내용 | 데이터 출처 |
|--------|------|------------|
| **메인 (/)** | 히어로 + 주간미션 + 도구모음 + 인사이트 | missions, tools, insights |
| **주간 미션 (/archive)** | 주차별 멤버 과제 카드 + 상세 모달 | missions 폴더의 .md 파일 |
| **인사이트 (/insights)** | 멤버들의 발견과 아이디어 | insights 폴더 |
| **도구 모음 (/tools)** | 실전 검증 도구·스킬셋 목록 | tools 폴더 |
| **갤러리 (/gallery)** | 배포된 프로젝트 썸네일 | gallery 폴더 |
| **AI 분석 (/analysis)** | 주차별 자동 분석 리포트 | analysis 폴더 |
| **팀소개 (/team)** | 멤버 프로필 + 한줄소개 | members 폴더 |

**핵심**: 웹사이트와 어드민은 **같은 프로젝트, 같은 데이터**를 공유한다.
어드민이 파일을 수정하면 → 웹사이트가 재빌드되면서 반영.

---

## 3. 어드민 동작 방식 (4단계)

![어드민구조_3.png](/images/어드민구조_3.png)

어드민 URL: **aaa-homepage.vercel.app/admin**

### 어드민이란?
노션처럼 웹 브라우저에서 콘텐츠를 수정할 수 있는 관리 페이지.
코드를 몰라도, 터미널을 몰라도, **폼에 내용을 채우고 저장 버튼만 누르면** 된다.

### 쉽게 말하면
- 어드민에서 "저장"을 누르면, **GitHub에 자동으로 커밋**이 생긴다
- GitHub에 커밋이 생기면, **Vercel이 알아서 사이트를 다시 빌드**한다
- 1~2분 후 **웹사이트에 반영**된다

## 5. 핵심 기능 3가지

![어드민구조_5.png](/images/어드민구조_5.png)

### 5-1. 태그 관리 시스템

도구모음의 "카테고리"와 인사이트의 "키워드"를 **노션 다중선택**처럼 관리할 수 있다.

- **칩 클릭**: 선택/해제
- **X 버튼**: 태그 삭제 (hover 시 나타남)
- **새 태그 입력**: 자유롭게 추가
- **인사이트 키워드**: 최대 3개 제한 → 키워드가 너무 분산되지 않도록

### 5-2. 마크다운 본문 에디터

분석/제안 탭에서는 **frontmatter(메타정보) + 본문(내용)**을 함께 편집한다.

- 노션 스타일 툴바: H2, H3, 굵게, 목록, 인용, 코드블록
- 텍스트를 드래그해서 굵게 만들기 가능
- 모달이 넓어져서 편집 공간 충분

### 5-3. 이미지 업로드

갤러리 썸네일, 멤버 프로필 사진을 어드민에서 직접 업로드.

- 파일 선택 또는 URL 붙여넣기
- GitHub API로 레포에 직접 업로드
- 업로드 후 즉시 프리뷰 표시

## 7. 레포지토리 구조

3개의 레포가 각각 역할을 나눠서 돌아간다.

### 전체 레포 관계

```
selfishclub-all/aaa (Obsidian Vault — 콘텐츠 원본)
       │
       ├──→ selfish-aaa-site-astro (Astro 정적 사이트 — 웹사이트)
       │         sync-content.sh로 콘텐츠 동기화
       │
       └──→ aaa-admin (Next.js — 어드민 페이지)
                 GitHub API로 콘텐츠 직접 수정
```

### 1) selfish-aaa (Obsidian Vault)

```
selfish-aaa/
├── 00_missions/            ← 주차별 멤버 과제
│   ├── Week_01/
│   ├── Week_02/
│   └── ...
├── 02_insights/            ← 멤버들의 발견·아이디어
├── 05_meetings/            ← 주차별 회의록
├── 06_sharing/             ← 발표 자료 (지금 이 문서)
├── _analysis/weekly/       ← AI 분석 리포트
├── _status/                ← 진행현황 대시보드
└── _meta/                  ← 명령어 가이드 등
```

### 2) selfish-aaa-site-astro (Astro 웹사이트)

```
selfish-aaa-site-astro/
├── src/
│   ├── content/            ← 마크다운 콘텐츠 (vault에서 동기화)
│   │   ├── analysis/       → /analysis 페이지
│   │   ├── gallery/        → /gallery 페이지
│   │   ├── insights/       → /insights 페이지
│   │   ├── members/        → /team 페이지
│   │   ├── missions/       → /archive 페이지
│   │   ├── proposals/      → /proposals 페이지
│   │   └── tools/          → /tools 페이지
│   ├── pages/              ← 각 페이지 라우트
│   ├── layouts/            ← 레이아웃 컴포넌트
│   └── styles/             ← 글로벌 CSS
├── public/images/          ← 이미지·영상 에셋
└── astro.config.mjs
```

### 3) aaa-admin (Next.js 어드민)

```
aaa-admin/
├── app/                    ← Next.js App Router
│   ├── gallery/            ← 갤러리 관리
│   ├── members/            ← 멤버 관리
│   ├── tools/              ← 도구 관리
│   ├── insights/           ← 인사이트 관리
│   ├── analysis/           ← 분석 관리 (마크다운 에디터)
│   ├── proposals/          ← 제안 관리 (마크다운 에디터)
│   ├── missions/           ← 미션 관리
│   ├── deploy/             ← 배포 트리거
│   ├── api/                ← API 라우트 (auth, tags, members)
│   └── login/              ← 인증
├── components/             ← 공통 컴포넌트
│   ├── markdown-editor.tsx ← 마크다운 에디터
│   ├── tag-input.tsx       ← 태그 관리 UI
│   ├── content-form.tsx    ← 콘텐츠 폼
│   ├── search-filter.tsx   ← 검색·필터
│   └── sidebar.tsx         ← 사이드바
├── actions/                ← 서버 액션 (GitHub API 호출)
├── lib/                    ← 유틸리티
└── middleware.ts           ← 인증 미들웨어
```

### 배포 구조

| 레포 | 배포 대상 | 배포 방식 |
|------|----------|----------|
| selfish-aaa | — | Obsidian vault (로컬) |

## 정리

> **"Obsidian에 쓰면 웹사이트가 된다"**가 핵심 철학이고,
> **어드민은 코드를 모르는 사람도 수정할 수 있게 만든 GUI**다.
> 서버 없이, 데이터베이스 없이, GitHub + Vercel만으로 돌아간다.
