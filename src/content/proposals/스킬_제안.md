## 상세 제안

### 제안 1: 원소스 멀티유즈 스킬 (`/multiuse`)
- **제안자**: AI (에밀리, 다니, 띵끄의 과제에서 도출)
- **유형**: Claude Code 스킬
- **난이도**: 중급
- **관련 개념**: 원소스 멀티유즈, 콘텐츠 자동화, 멀티채널
- **설명**: URL 하나를 입력하면 인스타 캐러셀 텍스트 + 스레드 포스트 + 링크드인 포스트 + 블로그 초안을 한 번에 생성. 에밀리는 8채널 20개 콘텐츠를, 다니는 3채널을 이미 구현 → 공통 스킬로 통합하면 모든 멤버가 사용 가능
- **기대 효과**: 콘텐츠 제작 시간 90% 단축
- **추천 멤버**: 에밀리, 다니, 띵끄
- **참고**: [[Week_02_에밀리]], [[Week_02_다니]], [[Week_02_띵끄]]

### 제안 2: 인스타 카드뉴스 생성 스킬 (`/cardnews`)
- **제안자**: AI (오웬의 과제에서 도출)

- **기대 효과**: 카드뉴스 제작을 10분 이내로 단축
- **추천 멤버**: 오웬, 다니
- **참고**: [[Week_02_오웬]]

### 제안 7: 하네스 도입 자가진단 스킬 (`/harness-audit`)
- **제안자**: AI (Week 06 분석 — 다니, 다다, 흐민의 과제에서 도출)
- **유형**: Claude Code 스킬
- **난이도**: 중급
- **관련 개념**: Legibility · Mechanical Enforcement · Verification Loops · Docs as Map, OpenAI 하네스 엔지니어링 4축
- **설명**: 프로젝트 경로를 입력하면 현재 하네스 상태를 4축 기준으로 진단하고 Phase 1~4 도입 로드맵을 생성. 다니가 인스타 캐러셀 에디터에 설계한 4-Phase(문서 구조화 → Husky 린터 → Puppeteer 시각 회귀 → ADR + Doc-gardening)와, 다다의 AAA 스터디 하네스 표, 흐민의 Selforge 다층 구조를 공통 체크리스트로 통합
- **기대 효과**: "바이브 코딩" 상태인 프로젝트를 짧은 시간 안에 **코드로 강제되는 규칙**이 깔린 시스템으로 전환. 실수를 린터로 축적하는 자기 개선 루프 정착
- **추천 멤버**: 다니, 다다, 흐민, 비비안
- **참고**: [[Week_06_다니_submit]], [[Week_06_다다_submit]], [[Week_06_흐민_submit]]

### 제안 8: 관점 전환 프롬프트 스킬 (`/perspective-shift`)
- **제안자**: AI (Week 06 분석 — 에밀리의 과제에서 도출)
- **유형**: Claude Code 스킬
- **난이도**: 초급
- **관련 개념**: 하네스 엔지니어링 시즌2 — "같은 방향 3번 실패 = 방향이 틀렸다는 신호"
- **설명**: 현재 진행 중인 대화를 분석해 "같은 관점에서의 반복 시도"가 감지되면 발상 전환 질문을 자동 제안. 에밀리가 실전에서 검증한 `"하나씩 꼭 이렇게 해봐야 아는 거야?"` + `"다른 관점에서 생각해봐"` 패턴을 트리거 기반 슬래시 명령으로 만들어, 기술을 몰라도 AI의 삽질을 끊을 수 있는 툴로 제공. fetch → axios → helpers → API 같은 4연속 실패를 staticData 플래그 한 줄로 해결한 사례가 실증
- **기대 효과**: 비기술자 멤버가 AI와의 페어 프로그래밍에서 기술 몰라도 품질을 통제할 수 있는 진입점. 디버깅 시간 크게 단축
- **추천 멤버**: 에밀리, 비비안, 다다(운영자 톤으로 확장)
- **참고**: [[Week_06_에밀리_submit]]

### 제안 9: 작업 로그 롤업 스킬 (`/worklog-rollup`)
- **제안자**: AI (Week 06 분석 — 비비안의 과제에서 도출)
- **유형**: Claude Code 스킬
- **난이도**: 중급
- **관련 개념**: "왜 → 한 일 → 삽질과 배움" 포맷, 날짜별 로그, 배움 한 줄
- **설명**: 지정 기간의 git log, PR 코멘트, 작업 로그를 긁어 비비안이 셀피쉬클럽 v4.0 로그에서 증명한 **날짜별 "왜 / 한 일 / 삽질과 배움" + `→ 배움:` 한 줄** 포맷으로 자동 정리. 한 주 단위·한 달 단위로 롤업해 발표 자료나 링크드인 글의 재료로 바로 쓸 수 있게 함. `/worklog` 스킬(개별 기록)과 쌍을 이루는 상위 정리 도구
- **기대 효과**: 주간 회고·분석·발표 준비·링크드인 소재 발굴에 드는 시간을 수 시간 → 10분으로 단축. 삽질 로그가 저절로 팀 자산으로 축적
- **추천 멤버**: 비비안, 에밀리, 다다
- **참고**: [[Week_06_비비안_submit]], [[Week_06_에밀리_submit]]

### 제안 10: Watchdog 러너 템플릿 (`watchdog-runner`)
- **제안자**: AI (Week 06 분석 — 흐민의 과제에서 도출)
- **유형**: CLI 도구 (Bash + 서비스 템플릿)
- **난이도**: 고급
- **관련 개념**: `while true; do ... done` 무한 루프 러너, pending update 카운트 감시, 교착 감지 및 세션 재시작
- **설명**: 흐민의 Selforge에서 검증된 `sullivan-runner.sh` + `watchdog.sh` 페어를 범용 템플릿으로 패키징. Claude Code 세션이 죽거나 교착되어도 자동으로 되살리는 구조를, 채널 입력(Telegram/Slack/etc.) pending count를 감지하는 표준 훅과 함께 제공. **"모델이 죽어도 하네스는 안 죽는다"** 원칙의 레퍼런스 구현
- **기대 효과**: 장기 실행이 필요한 개인 AI 어시스턴트·24시간 에이전트·스케줄 기반 자동화 프로젝트에서 재시작/복구 로직을 매번 새로 짜지 않아도 됨
- **추천 멤버**: 흐민, 오웬(Mac mini OpenClaw 실험과 연결), 다니
- **참고**: [[Week_06_흐민_submit]], [[Week_06_오웬_submit]]
