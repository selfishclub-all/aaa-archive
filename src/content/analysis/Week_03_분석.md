## 이번 주 핵심 발견

**자동화의 완성은 파이프라인이 아니라 설계 철학이다.**

이번 주 제출물 6개를 관통하는 공통점은 하나였다. "무엇을 자동화했느냐"보다 "어떤 구조로 설계했느냐"가 결과물의 깊이를 결정했다. URL 하나로 8개 채널 20개 이상의 콘텐츠를 자동 생성한 CRM 파이프라인, 매주 월·목 12시 38분에 인스타그램 카드뉴스를 자동 업로드하는 스케줄러, 사용자에게 정보 대신 질문을 던지는 퍼스널 AI까지 — 모두 기능이 아니라 원칙에서 출발한 프로젝트였다.

## 주요 트렌드

### 1. 에이전트 팀 구조가 표준이 되고 있다

단일 AI 프롬프트가 아닌, 역할별 에이전트가 파이프라인으로 연결되는 구조가 이번 주 과반수 프로젝트에서 채택됐다. 한 멤버는 director → data-collector → copywriter → media-ops → dispatcher로 이어지는 6단계 에이전트 팀을 설계해 "CRM 시작해줘 {URL}" 한 마디로 알림톡 9종·카플친 배너 3종·오픈채팅 카피 9개·인스타 캐러셀·UTM 21개를 한번에 처리했다. 또 다른 멤버는 research → generate → render → upload를 파일 단위로 모듈화하여 디자인만 바꾸고 싶을 때 `generate.py` 하나만 수정하면 되는 구조를 만들었다. 퍼스널 AI를 설계한 멤버는 capture-agent, morning-agent, reflection-agent, insight-agent로 하루의 흐름을 나눴다. 공통 원칙은 하나다. 모듈이 작을수록 고치기 쉽고, 역할이 명확할수록 전체가 강해진다.

### 2. 플랫폼 정책이 코드보다 더 큰 벽이었다

이번 주 삽질 리포트에서 반복적으로 등장한 단어는 "차단"이었다. 인스타그램 공식 API는 신규 계정에서 비즈니스 인증·앱 심사·공개 URL 요건으로 막혔고, 비공식 라이브러리로 우회해도 신규 계정은 24~48시간의 유예기간이 필요했다. SOLAPI 알림톡 발송은 API 필드 구조가 SDK 문서와 달라 네 번의 시도 끝에 해결됐다. Threads API 토큰 수명은 기본 1시간이었다. Python 환경 버전 불일치로 instagrapi 버전을 다운그레이드해야 했고, .env 특수문자 처리에서도 한 번 더 막혔다. 한 멤버의 회고가 이 주를 정확히 요약했다. "파이프라인 코드는 하루 만에 다 만들었다. 로그인 하나 때문에 계속 삽질했다."

### 3. CLAUDE.md가 프로젝트의 실제 두뇌다

여러 멤버가 같은 발견을 각자의 언어로 기록했다. 기존 Webflow 사이트를 9단계로 분석한 뒤 CLAUDE.md를 완성하고 배포 일정을 ver 1~3으로 확정한 프로젝트, PRD 전체를 CLAUDE.md에 담아 에이전트 팀 전체에 맥락을 주입한 CRM 프로젝트, tone-guide.md를 살아있는 문서로 설계해 대화하면서 계속 업데이트하는 퍼스널 AI 모두 같은 교훈을 공유했다. "중요한 규칙은 자동으로 로드되는 파일에 넣어야 한다. 별도 파일은 명시적으로 참조하지 않으면 존재하지 않는 것과 같다." CLAUDE.md는 AI에게 건네는 설명서가 아니라, 프로젝트의 기억 자체다.

## 이번 주 독특한 시각

### "피드가 호기심의 주도권을 가져간다"

정보를 더 주는 AI가 아니라, 질문을 던지는 AI를 설계한 멤버가 있었다. SNS를 열면 내가 궁금해서 찾아본 것보다 피드가 보여줘서 소비한 것들이 훨씬 많아진다는 문제의식에서 출발한 프로젝트였다. 해결책은 콘텐츠 필터링이 아니라 매일 나만의 질문을 되찾는 AI 에이전트였다. 아침 브리핑은 어제 포착한 생각을 연결하고 오늘의 질문을 하나 던진다. 생각이 떠오르면 텔레그램으로 흘려보내면 에이전트가 분류·연결·저장한다. 자기 전 회고는 오늘의 발견과 내일을 위한 질문으로 마무리된다. 이 프로젝트가 이번 주 MVP 6표를 받은 이유는 완성도보다 질문의 방향이 달랐기 때문이다.
