## 이번 주 핵심 발견

**자동화의 단위는 채널이 아니라 워크플로우다.**

이번 주 과제에서 대부분의 멤버가 콘텐츠 자동화를 시도했다. 흥미로운 점은, 성공적인 결과물의 공통점이 "인스타 자동화" 같은 채널 단위가 아니라 **URL→분석→생성→배포**라는 워크플로우 전체를 자동화한 것이라는 점이다. 채널 하나만 자동화하면 채널 하나만 된다. 워크플로우를 자동화하면 채널 추가가 쉬워진다.

---

## 주요 트렌드

### 1. 원소스 멀티유즈의 폭발

URL 하나에서 여러 채널 콘텐츠를 자동 생성하는 패턴이 이번 주의 핵심 주제였다. 과반수의 멤버가 "하나의 소스에서 여러 아웃풋"을 구현했다.

- 공유회 URL 하나로 알림톡 9종 + 카플친 배너 3종 + 오픈채팅 카피 9개 + 인스타 캐러셀 + 이메일 + UTM 21개를 자동 생성
- YouTube URL 하나로 인스타 캐러셀 + 스레드 + 링크드인 포스트 동시 생성
- 콘텐츠 마케팅 사이트 + Vercel 자동배포 + YouTube API 연동

### 2. 에이전트 팀 설계의 등장

단일 AI가 아닌, 역할별로 분담하는 멀티에이전트 팀 구조가 처음으로 등장했다. 6개 에이전트가 파이프라인으로 연결되는 구조:

```
director → data-collector → timeline-planner → copywriter → media-ops → dispatcher
```

이 패턴은 복잡한 워크플로우를 모듈화하여 각 에이전트가 독립적으로 동작하면서도 전체가 유기적으로 연결되는 아키텍처다.

### 3. CLAUDE.md = 프로젝트의 두뇌

여러 멤버가 CLAUDE.md(프로젝트 맥락 파일)의 중요성을 발견했다. 프로젝트의 전체 맥락을 하나의 파일에 담아두면 AI와의 협업 품질이 극적으로 향상된다. 특히:

- 기존 사이트를 9단계로 분석한 후 CLAUDE.md를 작성
- PRD를 CLAUDE.md에 통째로 넣어서 에이전트 팀에게 맥락 제공
- "어디까지 했더라?"가 사라지는 경험

### 4. 비개발자도 실전 프로젝트를 완성할 수 있다

프레임워크 없이 순수 HTML/CSS/JS만으로 프리미엄 웹 제안서를 하루 만에 완성한 사례가 있었다. Claude Code와 실시간 페어 프로그래밍으로 기획→디자인→개발→배포를 혼자 해냈다. 코딩 경험이 적어도 CLAUDE.md를 잘 작성하면 복잡한 프로젝트도 진행 가능하다는 증명.

## 이번 주 독특한 시각

### "AI에게 답을 구하지 않기로 했다"

대부분의 멤버가 AI를 "생산성 도구"로 활용한 반면, 한 멤버는 완전히 다른 접근을 했다. "피드가 호기심의 주도권을 가져간다"는 문제의식에서 출발해, **정보를 더 주는 AI가 아니라 질문을 던져서 생각을 확장하는 AI**를 설계했다.

Anne Sullivan(헬렌 켈러의 선생님)의 교육 철학을 AI 설계 원칙으로 적용한 독창적 접근:
- 살아있는 호기심 — 정해진 커리큘럼 대신 관심사 중심
- 생각하게 만든다 — 빠른 답 대신 질문 먼저
- 틀을 유연하게 — tone-guide는 대화하면서 계속 업데이트
